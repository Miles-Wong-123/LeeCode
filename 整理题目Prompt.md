# LeetCode 题目整理 Prompt

请按照本仓库已有题目的整理方式，整理我提供的 LeetCode 题目。

## 整理目标

为每道题创建一个独立目录，目录中包含：

- `problem.md`：题目描述、示例、解释。
- `Solution.java`：Java 解法，文件顶部包含中文解题思路、时间复杂度和空间复杂度。

## 目录规则

1. 先根据题目所属算法类型，选择根目录下对应分类文件夹。
2. 如果分类文件夹已存在，直接放入该分类下。
3. 题目目录命名格式为：`题号. 中文题名`。
4. 例如：

```text
02-two-pointers/11. 盛最多水的容器/problem.md
02-two-pointers/11. 盛最多水的容器/Solution.java
```

## `problem.md` 格式

保持简洁，结构参考：

```markdown
# 题号. 中文题名

题目描述正文，关键变量用反引号包裹，例如 `nums`、`target`。

## 示例 1

输入：`...`

输出：`...`

解释：...

## 示例 2

输入：`...`

输出：`...`
```

整理题面时要注意：

- 保留题目的核心描述、限制说明和示例。
- 示例标题使用 `## 示例 1`、`## 示例 2`。
- 输入、输出、解释之间用空行隔开。
- 行文使用中文，关键代码变量、数组、字符串使用 Markdown 反引号。

## `Solution.java` 格式

文件顶部必须包含中文块注释，格式参考：

```java
/*
解题思路：
1. ...
2. ...
3. ...

时间复杂度：O(...)。
空间复杂度：O(...)。
*/
class Solution {
    public 返回类型 方法名(参数列表) {
        // 解法代码
    }
}
```

代码要求：

- 使用 LeetCode 标准的 `class Solution`。
- 方法签名必须与题目要求一致。
- 优先使用清晰、主流、容易复盘的解法。
- 变量命名要表达含义，例如 `left`、`right`、`maxArea`、`numSet`。
- 只在必要时添加代码内注释，避免重复解释显而易见的代码。
- 如果需要引入类，放在块注释之后、`class Solution` 之前。

## 解题思路要求

解题思路要写清楚：

1. 使用了什么核心方法或数据结构。
2. 为什么这个方法可行。
3. 每一步如何更新状态或结果。
4. 最后返回什么。

复杂度格式统一为：

```text
时间复杂度：O(n)。
空间复杂度：O(1)。
```

## 执行流程

当我给出一道题时，请按以下流程处理：

1. 查看仓库已有题目的格式，特别是同类目录或 `01-arrays-hashing` 下的样例。
2. 判断题目分类，创建对应题目目录。
3. 创建 `problem.md`，按仓库格式整理题面。
4. 创建 `Solution.java`，写出 Java 解法和中文思路注释。
5. 使用 `javac Solution.java` 编译验证。
6. 最后告诉我新增了哪些文件、采用了什么解法、是否编译通过。
